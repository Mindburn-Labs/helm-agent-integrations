# AGENTS.md - helm-agent-integrations

This repository contains HELM-compatible integration examples and adapters.
It does not define HELM conformance, HELM verdict semantics, receipt semantics,
or EvidencePack verification rules. Those remain owned by
`helm-ai-kernel`.

## Working Rules

- Keep adapters thin: normalize a framework tool call, submit a HELM preflight,
  dispatch only on `ALLOW`, and return decision/receipt metadata.
- Do not claim partnership, certification, endorsement, or conformance unless
  a source-owned artifact proves it.
- Sample receipts and EvidencePacks must be generated by
  `scripts/generate_samples.py`; do not hand-edit generated samples.
- Demos must show at least one non-dispatching path: `DENY` or `ESCALATE`.
- Public copy should say "HELM-compatible example" or "optional HELM boundary",
  not "official integration" unless the upstream project accepts it.

## Validation

Run:

```bash
make validate
```

If dependency installation is unavailable, run the focused checks that do not
need network:

```bash
python3 scripts/generate_samples.py --check
python3 scripts/verify_samples.py
python3 -m unittest discover packages/python/helm_tool_wrapper/tests
```
