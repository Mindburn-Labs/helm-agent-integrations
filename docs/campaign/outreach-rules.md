# Outreach Rules

Ship working code before outreach.

For each target:

1. Build the smallest adapter or example.
2. Include a dangerous-action demo.
3. Include policy material.
4. Include receipt and EvidencePack samples.
5. Run validation.
6. Open one small issue or PR.
7. Offer to maintain externally.

Do not:

- fake community activity
- create sockpuppets
- astroturf forums
- imply partnership without acceptance
- spam maintainers
- attack projects for lacking safety

Use "HELM-compatible example" until upstream acceptance exists.
